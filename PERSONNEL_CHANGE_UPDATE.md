# 人員異動表單更新總結

## 🎉 更新完成項目

### ✅ 1. 新增人員表單邏輯更新

#### 變更內容
- **職級**：改為下拉選單，預設 NAG
  - 選項：NAG, AG, ASA, SAS, UM
- **主管 AGCODE**：新增查詢功能
  - 輸入主管 AGCODE
  - 點擊搜尋按鈕查詢
  - 自動驗證主管職級（必須是 ASA/SAS/UM）
- **主管姓名**：自動帶入（唯讀）
- **單位**：自動帶入主管的單位（唯讀）

#### 驗證規則
- 必須輸入姓名
- 必須查詢並選擇主管
- 主管必須是 ASA/SAS/UM 職級
- 單位必須自動帶入

---

### ✅ 2. 職級/單位異動表單更新

#### 變更內容
- **新職級**：改為下拉選單
  - 選項：NAG, AG, ASA, SAS, UM
  - 自動偵測是否為主管職級

#### 主管職級特殊功能
當選擇 ASA/SAS/UM 時：
1. **顯示主管欄位**
   - 可輸入主管 AGCODE 並查詢
   - 自動帶入主管姓名

2. **單位選擇方式**（三選一）
   - ✅ **選擇現有單位**：從下拉選單選擇
   - ✅ **新增單位**：勾選後可自行輸入
   - ✅ **單位無異動**：勾選後保持原單位 ⭐ 新增

---

### ✅ 3. 問題修復

#### 修復：新單位選單無法顯示
**原因**：使用了不存在的 `google.script.run.getAllData()` API

**解決方案**：改用 fetch API
```javascript
fetch(GAS_API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ action: 'getAllData' })
})
```

## 📋 使用流程

### 新增人員流程
```
1. 輸入 AGCODE 查詢
2. 系統確認無此人員
3. 填寫姓名
4. 選擇職級（預設 NAG）
5. 輸入主管 AGCODE → 點擊搜尋
6. 系統驗證主管職級並自動帶入姓名和單位
7. 送出申請
```

### 職級/單位異動流程

#### 一般職級（NAG/AG）
```
1. 選擇新職級
2. 從下拉選單選擇新單位
3. 填寫異動原因（選填）
4. 送出申請
```

#### 主管職級（ASA/SAS/UM）
```
1. 選擇新職級（ASA/SAS/UM）
2. 系統顯示主管欄位和單位選項
3. 輸入主管 AGCODE → 查詢（選填）
4. 選擇單位方式：
   - 方案 A：從現有單位選擇
   - 方案 B：勾選「新增單位」自行輸入
   - 方案 C：勾選「單位無異動」保持原單位
5. 填寫異動原因（選填）
6. 送出申請
```

## 🎨 互動邏輯

### 單位選擇互斥邏輯
```javascript
// 三個選項互斥
if (勾選「單位無異動」) {
    取消勾選「新增單位」
    隱藏所有輸入框
    newGroup = oldGroup
}

if (勾選「新增單位」) {
    取消勾選「單位無異動」
    顯示文字輸入框
    隱藏下拉選單
}

if (都不勾選) {
    顯示下拉選單
    隱藏文字輸入框
}
```

## 📊 使用案例

### 案例 1：NAG → AG（一般職級）
- 職級：NAG → AG
- 單位：從下拉選單選擇

### 案例 2：AG → ASA（升主管）
- 職級：AG → ASA
- 主管：輸入主管 AGCODE
- 單位：選擇現有/新增/保持原單位

### 案例 3：ASA → SAS（主管升職，單位不變）
- 職級：ASA → SAS
- 主管：輸入主管 AGCODE
- 單位：勾選「單位無異動」✨
- 結果：職級變更，單位不變

### 案例 4：ASA → ASA（同職級調單位）
- 職級：ASA → ASA（不變）
- 單位：從下拉選單選擇新單位
- 結果：職級不變，單位變更
